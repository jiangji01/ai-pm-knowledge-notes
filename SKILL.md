# AI PM 知识点笔记（三面 Skill）

用户在整理 **单个知识域或多个子知识点** 时，默认按本 Skill 产出 Markdown 笔记：**同一套三块镜子，全文一致。**

**如何将本 Skill 装进 Cursor / Claude Code**：见同级 [README.md](README.md)。

---

## 1. 落盘约定

| 项 | 规则 |
|:---|:---|
| **路径** | **本项目**：写入 iCloud `AI Memo` 下 `AI产品应知应会/{知识点}.md`（本机路径见仓库根目录 `OBSIDIAN.md`）。其它工程仍可从对话指定目录。 |
| **文件名** | **仅用语义化的知识点名作文件名**，如 `大模型核心技术原理.md`、`RAG检索增强.md`。禁止使用 `JD一条拆`、拆解、备忘等后缀。 |
| **双链 / 索引** | 若库内另有 MOC：`[[知识点篇名]]`。用户未要求时不要改 unrelated 文件的链接。 |

---

## 2. 三面界线（必须先对齐再写）

| 角度 | 回答的问题 | 禁止 |
|:---|:---|:---|
| **① 知识本身是什么** | 教材/综述/官方文档语境下公认的 **定义、组件、因果关系、术语边界**。 | 不出现「我们觉得」或岗位话术；不写未经验证的排名、「最强」口嗨。 |
| **② AI PM 如何消化认知** | PM **不做训练推导**前提下，内化成的 **心智模型、检查清单、与算法/工程对齐时要问出口的句式**。 | 不写具体 PRD 需求文档全文。 |
| **③ 应用层如何理解与实践** | 落到 **产品形态、链路、指标、灰度、成本与体验、门禁** 的可操作理解。 | 避免与 ② 重复：② 写 **判断原则**，③ 写 **落地形态与验收**。 |

单篇可以只讲 **一个**大知识点（文内用「一、二、三」节拆子题），或 **一章总览 + 多节子知识点**；**每个子知识点** 下仍须具备 **①②③** 三级标题（或在该节内用加粗小标题 `**①**`…，但优先用 `###` 以方便大纲导航）。

---

## 3. 正文模板（复制后填空）

```markdown
# {知识点标题}

> 来源：{可选：JD原句 / 书籍章节 / 课程 / 自拟}  
> 笔记结构：**① 知识本身（公信拆解）｜② AI PM 内化认知｜③ 应用层理解与实践。**  
> 标签建议：`#AI产品` + 领域标签

---

## {可选：总览或小标题}

- {这一章在考什么 / 知识边界 / 读完能答什么 — 2～5 条，非必须}

---

## {一、子知识点 A 或 单一主题}

### ① 知识本身是什么

- …

### ② AI PM 如何消化认知

- …

### ③ 应用层如何理解与实践

- …

---

## 附：{可选：面试口述 30s / 被深挖时的回扣点}

- …

---

## 口述与深挖（当用户明确要求「口述 / 深挖 / 演练」时追加，可独立成小节）

### 口述 · 90 秒稿（提纲式 Bullet，不写散文）

- **第一句人设 + 题目的关键词**（≤10 秒）
- **3 枚要点子弹**：每枚 = **论断 + 1 个可核实例子 / 类比**（可来自正文 ③）
- **一句收尾**：面试官若追问，我准备往「② 的判别」还是「③ 指标」引

### 深挖 · 自问 6 题（自己答不出就回去补正文）

每条留 **半行答案提示** 即可：谁 / 多少 / 何时 / 失败怎么办 / 与谁分工 / 如何验收。

文内可加二级标题：**《口述 · 90 秒（提纲）》**（bullet）；**《深挖 · 预埋追问（自问）》**（编号 6 条，条目后附简短提示）。

```

- **总览**节：适用于「一条 JD 含多个括号」类长文；独立小词条可省略总览。  
- **附 / 口述 / 深挖**：用户要面试口径时再加；「口述与深挖」可与正文同一次生成，也可第二遍迭代。

## 4. 执行清单（Writer 自检）

- [ ] 文件名 = 知识点名，无「拆解类」后缀  
- [ ] 每一块知识至少覆盖 ①②③；子主题不丢角  
- [ ] ① 可与教科/文档核对；忌拍脑袋「业界第一」  
- [ ] ② / ③ 分工清晰：原则 vs 落地  
- [ ] 若用户要 **口述演练**：补 **90 秒提纲 + 深挖自问**，并与个人项目经历预留接口（可写「此处接简历某项目」）
- [ ] 与个人面试话术材料互链仅限用户明确要求时

## 5. 参考示例

任选复杂知识域写一篇成稿（如「Transformer」「RAG」「预训练与微调」），或对已有 Markdown **套用第 4 节清单**逐项自检格式与三面完整度。

---

## 6. 我们与用户「深挖」时可协作的事

1. **第一遍**：按模板写 **①②③** 正文。  
2. **第二遍**（同文件追加）：**90 秒提纲 + 6 个预埋追问**，追问尽量 **指回简历里可举的项目**（由用户指定或从对话已知经历映射）。  
3. **第三遍**（用户侧）：口述录音 → 删减 bullet → 与 [[口述与深挖-工作流]] 中的自检表打分；下一轮再精炼。
